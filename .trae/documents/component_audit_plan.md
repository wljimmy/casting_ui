# 组件审计与手册完善计划

## 目标
逐一检查手册的每一个页面，检查CSS和JS里的每一个组件，确保都准确出现在手册里，并且包含使用说明、引用方法、是否涉及JS动态渲染等信息。

## 执行步骤

### 第一阶段：收集现有组件清单

#### 1.1 独立模组识别（作为整体介绍）
以下模组作为独立功能单元，不拆分内部CSS/JS，整体介绍：
- [ ] **MD编辑器** (`md.js` + `md.css`) - Markdown编辑器完整模组
- [ ] **主题管理器** (`theme-manager.js`) - 主题管理完整模组
- [ ] **颜色选择器** (`color-picker.js`) - 颜色选择器完整模组

#### 1.2 CSS组件清单收集（按类别归纳）
- [ ] 读取 `dev/modules/css/core.css`，提取所有组件类名
  - **容器类组件**：统一归纳到"容器"手册页介绍
  - **布局类组件**：统一归纳到"布局"手册页介绍
  - **工具类组件**：统一归纳到"工具类"手册页介绍
- [ ] 读取 `dev/modules/css/components.css`，提取所有组件类名
  - **按钮类**：统一归纳到"按钮"手册页介绍
  - **表单类**：统一归纳到"表单"手册页介绍
  - **数据展示类**：统一归纳到"数据展示"手册页介绍
  - **反馈类**：统一归纳到"反馈"手册页介绍
- [ ] 读取 `dev/modules/css/md.css` - 作为MD编辑器模组的一部分整体介绍

#### 1.3 JS组件清单收集（按功能归纳）
- [ ] 读取 `dev/modules/js/core.js`，提取所有功能/类
- [ ] 读取 `dev/modules/js/modal.js` - 弹窗功能模组
- [ ] 读取 `dev/modules/js/message.js` - 消息提示功能模组
- [ ] 读取 `dev/modules/js/image-zoom.js` - 图片放大功能模组
- [ ] 读取 `dev/modules/js/theme-manager.js` - 主题管理器模组（整体介绍）
- [ ] 读取 `dev/modules/js/color-picker.js` - 颜色选择器模组（整体介绍）
- [ ] 读取 `dev/modules/js/ui.js`，提取UI组件管理功能
- [ ] 读取 `dev/modules/js/md.js` - MD编辑器模组（整体介绍）
- [ ] 整理JS组件清单（函数/类名、功能描述、参数）

### 第二阶段：收集现有手册页面清单

#### 2.1 页面目录扫描
- [ ] 扫描 `pages/` 目录下所有子目录和页面
- [ ] 扫描 `test/` 目录下所有测试页面
- [ ] 整理现有手册页面清单（路径、标题、包含的组件）

#### 2.2 主菜单结构分析
- [ ] 读取 `index.html` 中的菜单结构
- [ ] 整理菜单层级和对应页面映射

### 第三阶段：对比分析

#### 3.1 创建对比表格
- [ ] 创建组件与手册页面对照表
- [ ] 标记每个组件的文档状态：
  - ✅ 已完整文档化（有手册页、有示例、有说明）
  - ⚠️ 部分文档化（有手册页但缺少说明/示例）
  - ❌ 未文档化（无手册页）

#### 3.2 缺失项识别
- [ ] 列出所有缺少手册页的组件
- [ ] 列出所有缺少测试页的组件
- [ ] 列出所有缺少菜单项的组件
- [ ] 列出文档不完整需要补充的组件

### 第四阶段：补充缺失内容

#### 4.1 补充手册页面

**独立模组手册页**（整体介绍，不拆分内部CSS/JS）：
- [ ] MD编辑器模组手册页
  - 功能概述
  - 完整使用示例
  - API方法说明
  - 配置参数
  - 事件回调
- [ ] 主题管理器模组手册页
  - 功能概述
  - 完整使用示例
  - API方法说明
- [ ] 颜色选择器模组手册页
  - 功能概述
  - 完整使用示例
  - API方法说明

**归纳类组件手册页**（按类别统一介绍）：
- [ ] 容器类组件手册页 - 统一介绍所有容器相关类
- [ ] 布局类组件手册页 - 统一介绍所有布局相关类
- [ ] 按钮类组件手册页 - 统一介绍所有按钮相关类
- [ ] 表单类组件手册页 - 统一介绍所有表单相关类
- [ ] 数据展示类组件手册页 - 统一介绍表格、列表、徽章等
- [ ] 反馈类组件手册页 - 统一介绍消息提示、加载等
- [ ] 工具类组件手册页 - 统一介绍间距、分割线等工具类

**单个组件手册页**（对于复杂独立组件）：
- [ ] 创建组件目录 `pages/{category}/{component}-container/`
- [ ] 创建 `index.html` 手册页面，包含：
  - 组件标题和简介
  - UI示例展示
  - 参数说明表格
  - 使用说明
  - 引用方法（代码示例）
  - 是否涉及JS动态渲染说明

#### 4.2 补充测试页面
对于每个缺失测试页的组件：
- [ ] 创建测试目录 `test/{component}/`
- [ ] 创建 `index.html` 测试页面，包含：
  - 基础功能测试
  - 边界情况测试
  - 交互测试（如适用）

#### 4.3 更新主菜单
- [ ] 在 `index.html` 中添加缺失的菜单项
- [ ] 更新页面映射 `pageMap`
- [ ] 添加适当的徽章标记（如 new）

### 第五阶段：验证与完善

#### 5.1 完整性验证
- [ ] 逐一验证每个组件都有对应的手册页
- [ ] 逐一验证每个组件都有对应的测试页
- [ ] 逐一验证每个组件都在菜单中有入口

#### 5.2 质量检查
- [ ] 检查所有手册页是否包含必需的章节：
  - UI示例
  - 参数说明
  - 使用说明
  - 引用方法
  - JS动态渲染说明
- [ ] 检查代码示例是否使用Prism高亮
- [ ] 检查样式是否统一

#### 5.3 版本更新
- [ ] 更新 `version.json` 中的模块版本
- [ ] 更新 `docs/logs/version.md` 添加更新日志
- [ ] 为修改的模块文件添加版本注释

### 第六阶段：最终提交

- [ ] 执行git提交
- [ ] 整理记忆索引

## 输出交付物

1. 组件清单（CSS + JS）
2. 手册页面清单
3. 缺失项报告
4. 新增/修改的手册页面
5. 新增/修改的测试页面
6. 更新后的菜单结构
7. 版本更新记录

## 注意事项

1. 严格遵循框架规范：
   - 使用框架内置样式，不添加内联样式
   - 代码示例使用Prism高亮
   - 文档结构统一

2. 保持代码简洁：
   - 不添加冗余代码
   - 使用语义化标签
   - 遵循现有命名规范

3. 版本管理：
   - 每次修改模块文件更新版本号
   - 记录详细的更新日志
