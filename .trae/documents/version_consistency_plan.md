# 文档与代码一致性检查及更新计划

## 任务目标
1. **检查文档内容与实际代码的一致性** - 确保文档描述的功能、API、用法与代码实现匹配
2. **检查版本号一致性** - 确保文档中引用的版本号与实际代码版本一致
3. **全面更新README文件** - 反映项目当前真实状态
4. **维护版本更新记录** - 与git提交历史保持一致

## 执行步骤

### 阶段一：收集当前项目状态
1. **读取package.json** - 获取项目主版本号、依赖、脚本
2. **扫描src/modules/css/** - 列出所有CSS模块及其实际功能
3. **扫描src/modules/js/** - 列出所有JS模块及其实际功能
4. **扫描src/examples/** - 了解现有示例
5. **扫描public/manual/** - 了解手册内容
6. **读取现有README.md** - 了解当前文档内容
7. **扫描docs/目录** - 了解现有设计文档

### 阶段二：文档内容一致性检查
1. **检查README.md中的目录结构**
   - 对比README描述的目录与实际目录结构
   - 标记缺失或多余的目录说明

2. **检查README.md中的功能列表**
   - 对比README描述的功能与实际代码功能
   - 标记已删除但未从文档移除的功能
   - 标记新增但未在文档说明的功能

3. **检查README.md中的API说明**
   - 对比API参数、返回值与实际代码
   - 标记已过时的API说明
   - 标记新增但未文档化的API

4. **检查docs/design/设计文档**
   - 对比设计规范与实际代码实现
   - 标记已实现但未记录的设计变更
   - 标记已变更但未更新的设计规范

5. **检查手册页面内容**
   - 对比手册中的组件用法与实际代码
   - 标记已变更但未更新的使用示例

### 阶段三：版本号一致性检查
1. **检查文档中引用的版本号**
   - README中提到的版本号
   - 手册页面中的版本号
   - 设计文档中的版本号

2. **对比实际代码版本号**
   - package.json版本号
   - 各模块文件头版本号

3. **标记版本号不一致的位置**

### 阶段四：更新文档内容
1. **更新README.md**
   - 修正目录结构描述
   - 更新功能列表（添加新功能，删除已移除功能）
   - 更新API说明（修正参数、返回值）
   - 更新使用示例（使用最新API和类名）
   - 更新版本信息
   - 添加最新重大变更说明

2. **更新docs/design/设计文档**
   - 更新与实际代码一致的设计规范
   - 添加新增组件的设计说明
   - 移除已删除组件的设计文档

3. **更新手册页面**
   - 修正组件使用示例
   - 更新类名（如CUI-前缀）
   - 更新API调用方式

### 阶段五：统一版本号
1. **更新所有模块文件头版本号**
   - CSS模块：core.css、container.css、layout.css等
   - JS模块：core.js、modal.js、menu.js等

2. **更新文档中引用的版本号**
   - README.md中的版本说明
   - 手册页面中的版本信息

### 阶段六：创建/更新版本日志
1. **创建docs/logs/CHANGELOG.md**
   - 按时间倒序记录所有版本变更
   - 每个版本包含：版本号、日期、变更类型(feat/fix/docs等)、详细变更列表
   - 与git提交历史保持一致

2. **版本日志内容组织**
   ```
   ## [0.3.0] - 2026-04-03
   ### Added
   - 新增布局系统：网页布局和应用壳布局
   - 新增CUI-grid网格系统
   
   ### Changed
   - 统一所有类名添加CUI前缀
   - 更新默认主题为墨黑酒红
   
   ### Removed
   - 移除旧的布局类名（无CUI前缀的）
   ```

### 阶段七：最终验证
1. **验证文档与代码一致性**
   - 随机抽查几个功能点，验证文档描述与代码一致

2. **验证版本号统一**
   - 确认所有模块版本号一致

3. **验证README完整性**
   - 确认README包含项目完整信息

4. **准备提交信息**
   - 按照git规范准备提交信息

## 预期产出
1. **更新后的README.md** - 反映项目当前真实状态
2. **更新后的设计文档** - 与实际代码实现一致
3. **更新后的手册页面** - 使用最新的API和类名
4. **docs/logs/CHANGELOG.md** - 完整的版本变更记录
5. **统一版本号的模块文件** - 所有模块版本号一致

- [ ] README功能列表完整准确
- [ ] README使用示例可正常运行
- [ ] 设计文档与代码实现一致
- [ ] 手册页面使用最新的类名和API
- [ ] 所有模块版本号统一
- [ ] 版本日志与git历史一致
