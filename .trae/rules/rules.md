# Casting 极简UI框架AI开发约束规范

（已更新：采用 Vite 标准目录结构 + 手册页目录规范，保留原有约束风格与逻辑）
核心目的：让用户向使用md一样使用Casting框架，无需学习新的语法或工具。让用户专注内容，做到除了业务内容，其他内容都由框架自动处理（如Casting.css、Casting.js）。同时保留用户自定义样式的能力（为了满足深度自定义的需求）。使用最简化引用，css零配置，UI互动直接方法调用（方法（{参数}））。
1. 框架定位与技术栈：定位为无业务功能的极简UI框架，适配TREA CN开发环境；技术栈仅限原生HTML5、CSS、JS（ES6+），引入Vite作为本地开发预览工具，仅用于开发环境热更新与本地服务运行，不参与代码编译、转译、压缩与构建打包，不修改任何源码语法；代码可直接浏览器打开运行，无需服务器/部署；仅在大版本迭代时手动打包，无自动打包流程，最终对外发布产物仅为合并后的Casting.css与Casting.js两个通用功能框架文件，框架文件不包含业务内容、测试/示例相关的内容或临时样式。
2. 目录结构：采用Vite标准工程目录结构，严格划分目录职能并隔离不混用；保留`src`作为框架核心源码目录、`public`作为静态资源目录、src目录`index.html`作为Vite入口示例页；**手册页统一存放在public/manual目录下，由src/index.html通过fetch方式加载**；同时保留测试目录test、发布目录dist、文档目录docs及项目配置文件；发布目录dist按版本号创建子目录，存储对应版本的Casting.css、Casting.js合并文件，保留所有历史版本归档，不删除过往版本；Vite相关配置文件、依赖目录仅存于根目录，不进入任何源码、测试、发布目录。
3. 开发目录（src）规范：对应原dev目录职能，存放模块化拆分的框架样式、脚本源码；src/modules存放所有框架核心功能模块；src/examples存放组件示例与功能演示页面；**手册页统一存放在public/manual目录下，由src/index.html通过fetch方式加载**；示例页、手册页均沿用现有模式，全量加载所有开发态模块文件，完整呈现框架全部功能、组件示例与使用说明；样式层面，View Transitions API、容器查询（@container）、clamp()/vw等弹性单位为框架通用能力，写入Casting.css；示例、手册专属布局/动画效果使用内联样式，不进入框架文件；视觉采用科技蓝（#165DFF）为主的流行配色，通过CSS变量统一管理样式，交互添加原生过渡动画（View Transitions API）；代码结构清晰（html/css/js分离），注释简洁，符合语义化规范，无冗余代码，content-visibility等性能优化属性为框架通用规则，仅在Casting.css中定义，不分散在示例、手册、测试页面。
   - **示例页/手册页样式规范（重要）**：示例页和手册页必须完全使用框架CSS类来展示样式，禁止使用内嵌`<style>`标签编写自定义样式；这样可以确保示例展示与实际应用的一致性；如果框架现有CSS类无法满足布局需求，应先扩展框架CSS类，而不是添加内嵌样式；
4. 静态资源目录（public）规范：存放无需处理的静态资源，包括SVG图标、字体、图片等；资源直接通过根路径引用，不参与任何编译处理，仅作为开发、示例与手册使用，不自动打包进框架核心文件。
5. 测试目录（test）规范：内为各独立功能对应的单测试页面，每个测试页面仅加载核心基础模块与目标测试功能模块，不引入无关模块；需包含原生JS编写的轻量自动测试逻辑，该逻辑仅内联写入test的HTML文件，绝不写入Casting.js/Casting.css，测试结果可视化展示；测试页面无JS内联/事件监听器使用约束，可按需选择内联JS（on\*属性）或事件监听器（addEventListener）；测试/示例/手册页面涉及的内容或临时样式需使用内联格式，禁止写入框架文件，且内联样式仅作用于自身局部，不得修改/覆盖框架CSS变量、全局类名。
6. 模块约束：内部按功能拆分样式与脚本模块，设置唯一核心基础模块，所有功能模块仅可依赖该核心模块，同级功能模块间禁止任何依赖；各模块独立维护版本信息、更新记录与API定义，迭代仅修改自身内容，不影响其他模块；模块采用原生ES6模块化语法，不使用Vite专属语法、别名、插件特性。
7. 引用规则：开发示例页、手册页全量引用src/modules下的模块文件；测试功能页仅引用核心模块+单一功能模块；Vite开发环境与原生直接运行环境引用路径完全一致；发布场景仅引用对应版本目录下的Casting.css与Casting.js合并文件。
8. 打包约束：手动打包仅合并`src/modules`内的框架模块，自动过滤测试代码、示例内容、手册内容、内联脚本/样式、Vite配置及依赖相关文件，保证发布包纯净，不包含任何非框架通用功能的冗余内容；dist目录仅存放最终合并后的框架文件，不存放任何开发环境、示例、手册相关内容。
9. 文档与交付：文档目录docs与根目录README需自动维护，同步各模块版本、更新信息、使用说明、API说明，模块变更后立即同步；交付内容包含可直接运行的完整代码，附极简使用说明（双击HTML即可运行），整体简练聚焦核心需求；每次操作后必须整理记忆并更新记忆索引，记忆维度包括：文件目录结构、各模块版本号、模块API、引用规则、CSS变量配置、禁止写入框架的内容清单，维持文件结构记忆库，方便后续查询回溯。
10. JS使用约束：
    - 约束目的：禁用事件监听器（addEventListener）仅为避免示例页面、手册子页面加载时产生监听错误，该约束仅对示例页面、手册页框架有效；
    - 基础规则：示例页面、手册页优先使用内联JS（on\*属性）实现业务交互，框架核心脚本（Casting.js）保持模块化，禁止内联；
    - 例外规则：若示例页面、手册子页面功能强制依赖事件监听器，可使用该方式，但需通过技术手段保证监听正常激活，且在更换页面后监听完全失效、从监听队列中清除，无残留。
11. Vite使用约束：
    - Vite仅用于本地开发预览与热更新，禁止执行vite build构建命令；
    - 不使用任何Vite插件、import.meta、路径别名等非原生语法；
    - 源码保持原生浏览器可直接运行标准，不做任何适配Vite的语法修改；
    - node_modules、vite.config.js、package.json等文件仅用于开发环境，绝不进入src、test、dist、docs目录；
    - Vite不参与框架最终打包与发布流程，发布包完全脱离Vite可独立运行；
    - **Vite开发环境路径规则**：src目录和public目录都是根目录，访问这些目录下的文件不需要加路径前缀；例如访问src/index.html只需写/index.html，访问public/manual/progress-container/index.html只需写/manual/progress-container/index.html；预览页面时不需要加src路径。

12. 命名空间管理规范：
    - 所有框架模块的全局变量必须使用 `window.CUI` 命名空间，避免全局变量冲突；
    - 模块暴露到全局时，必须先检查 `window.CUI` 是否存在，不存在则创建；
    - 禁止直接在 `window` 对象上添加全局变量，所有框架相关的全局变量都必须放在 `window.CUI` 命名空间下；
    - 命名空间命名应遵循驼峰命名法，与模块名保持一致。

13. 框架内联事件监听规范（仅限框架开发）：
    - 框架内部的监听器禁止使用常规 `addEventListener`，应使用框架封装的 `DOMObserver` 完成监听工作；
    - 使用 `DOMObserver` 可以防止监听队列过长，以及和用户手工监听出现的冲突；
    - 对于需要在HTML中直接调用的框架函数（如关闭按钮回调），禁止使用内联 `onclick` 属性，应通过 `DOMObserver` 监听元素变化后动态绑定；
    - 用户使用规范：用户可以随意使用内联 `onclick` 属性，框架需确保用户的内联调用与框架系统不冲突；
    - 框架应通过 `window.CUI` 命名空间隔离框架内部函数，确保用户自定义的全局函数不会覆盖框架功能。

14. 新模块建设文件要求：
    - **核心文件**：
      - `src/modules/js/模块名.js` - 模块的JavaScript实现
      - `src/modules/css/模块名.css` - 模块的CSS样式
    - **示例文件**：
      - `src/examples/模块名-examples.html` - 模块的功能示例和演示页面
    - **手册页**：
      - `public/manual/分类目录/模块名-container/index.html` - 模块的使用手册
    - **API文档**：
      - `docs/api/模块名.md` - 模块的API接口文档
    - 每次开始新模块建设时，必须先检查以上文件是否存在，若不存在则先创建再开始工作。
