## 四、框架核心设计原则

| 原则 | 说明 |
|------|------|
| **零配置引用** | 只有入 CSS 和 JS 文件即可使用，无需额外配置 |
| **data-* 属性驱动** | 用户通过 `data-*` 属性配置组件，框架自动渲染 |
| **原生标签优先** | 用户使用 `<menu>`、`<ul>`、`<li>` 等原生标签，框架自动增强 |
| **DOMObserver 管理事件** | 框架内部使用 `DOMObserver` 替代 `addEventListener`，防止监听泄露 |
| **命名空间隔离** | 所有全局变量放在 `window.CUI` 命名空间下 |
| **模块化架构** | 6 层模块依赖体系，通过生命周期调度器分阶段加载 |

## 五、模块加载顺序与依赖关系

```
ENV (env.js)
  │
CORE (core.js, dom-observer.js, scheduler.js)
  │
DOM_REGISTRY (template-engine.js, ui.js, overlay.js)
  │
INTERACTION (modal.js, message.js, menu.js, progress.js, image-zoom.js, color-picker.js, theme-manager.js)
  │
READY (form.js, input.js, location-data.js, idcard-validator.js)
```
