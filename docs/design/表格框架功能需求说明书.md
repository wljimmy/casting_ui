# 二、表格分类

表格框架分为以下两大类：

|分类|核心定位|主要特点|
|---|---|---|
|**展示类表格**|纯外观展示，不具备交互功能|时尚美观的表格外观，格式由用户完全自定义控制|
|**功能类表格**|具备完整交互功能的数据表格|支持冻结、合并、多种数据注入方式，满足复杂业务场景|

# 三、展示类表格功能需求

## 3\.1 核心定位

展示类表格是一种**纯展示型**的表格组件，不提供任何交互功能，仅用于数据的可视化展示。

## 3\.2 功能描述

1. **外观时尚化**：提供现代化、美观的表格样式，支持多种主题风格

2. **用户自定义格式**：表格的所有格式（颜色、字体、边框、间距、对齐方式等）完全由用户自行控制

3. **无交互功能**：不支持排序、筛选、编辑、冻结等任何交互操作

4. **纯渲染模式**：仅负责将用户传入的数据按照指定格式渲染展示

## 3\.3 适用场景

- 静态数据展示页面

- 报表打印导出

- 简单数据可视化

- 对样式要求高、无需交互的场景

# 四、功能类表格功能需求

## 4\.1 核心定位

功能类表格是一种**具备完整交互能力**的高级表格组件，支持复杂的数据操作和展示需求。

## 4\.2 内容注入方式

功能类表格支持**三种数据注入方式**，用户可根据场景灵活选择

|注入方式|功能描述|
|---|---|
|**标准表格框架注入**|通过表格框架提供的标准API接口，按照框架规范注入表格内容|
|**JSON格式传入**|直接传入JSON格式的数据对象，框架自动解析并渲染表格|
|**参数方式传入**|通过配置参数的方式，逐行逐列传入表格数据和配置|

## 4\.3 冻结功能

### 4\.3\.1 行冻结功能

支持指定任意行进行冻结，冻结行在垂直滚动时保持可见：

- **首行冻结**：冻结表格第一行（通常为表头）

- **前N行冻结**：支持冻结前2行、前3行等任意数量的开头行

- **汇总行冻结**：支持冻结表格底部的汇总行

- **多个汇总行冻结**：支持同时冻结底部多个汇总行

- **任意指定行冻结**：可通过行号参数指定任意行进行冻结

### 4\.3\.2 列冻结功能

支持指定任意列进行冻结，冻结列在水平滚动时保持可见：

- **首列冻结**：冻结表格第一列

- **最后一列冻结**：冻结表格最右侧的列（常用于操作列）

- **任意指定列冻结**：可通过列号参数指定任意列进行冻结

- **多列连续冻结**：支持连续多列同时冻结

### 4\.3\.3 组合冻结

支持行冻结与列冻结同时生效，形成固定的交叉区域：

- 左上角冻结区域：首行\+首列同时冻结

- 复杂冻结场景：前N行\+首N列组合冻结

## 4\.4 单元格合并功能

**重要规则**：单元格合并功能严格区分区域执行，不同区域有不同的合并权限

### 4\.4\.1 合并区域权限划分

|表格区域|是否允许合并|说明|
|---|---|---|
|**冻结列区域**|✅ 允许合并|冻结列内的单元格支持横向和纵向合并|
|**标题区域**|✅ 允许合并|各种标题行、表头区域完全支持单元格合并|
|**数据区域**|❌ 禁止合并|纯数据列区域**严格禁止**单元格合并，保证数据的纯净性|

### 4\.4\.2 合并功能详细说明

1. **横向合并（colspan）**：同一行内多个相邻列合并为一个单元格

2. **纵向合并（rowspan）**：同一列内多个相邻行合并为一个单元格

3. **交叉合并**：同时支持横向和纵向的矩形区域合并

4. **多级表头合并**：支持多层级表头的复杂合并结构

### 4\.4\.3 数据区域限制说明

数据区域（非冻结、非标题的纯数据列）之所以禁止单元格合并，是为了：

- 保证数据结构的规范性和一致性

- 便于数据导出、筛选、排序等操作

- 避免因合并导致的数据错乱和显示异常

- 符合关系型数据的设计原则

# 五、数据格式规范

## 5\.1 支持的数据类型

|数据类型|示例与说明|
|---|---|
|**文本类型**|短文本、长文本、大段描述文字，支持换行和特殊字符|
|**数字类型**|整数、小数、百分比、货币金额，支持格式化显示|
|**身份标识**|身份证号、手机号、工号等标准格式数据|
|**地址信息**|省份、城市、详细地址等地理位置信息|
|**链接类型**|图片URL、超链接、文件下载链接等|
|**日期时间**|日期、时间、日期时间范围等时间类型|

  ]
