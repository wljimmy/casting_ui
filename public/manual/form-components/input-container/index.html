Input 输入框

Input 模块提供注册表管理、值操作、表单批量操作、自动美化与数据验证功能。支持多种输入类型、自动图标渲染、标签布局、组合框样式和正则验证。

快速上手

只需为 input/textarea/select 添加 class="CUI-input"，框架自动完成渲染、注册和生命周期管理。

<input type="text" name="myInput" class="CUI-input" placeholder="文本输入">
<input type="password" name="myPassword" class="CUI-input" placeholder="密码输入">
<input type="email" name="myEmail" class="CUI-input" placeholder="邮箱输入">

数据验证（Validation）

验证规则表

使用 data-validate 属性声明验证规则，框架在失焦（blur）时自动验证，输入（input）时自动清除错误状态：

规则名称 正则表达式 默认错误提示
idcard/^\d{17}[\dXx]$/身份证号格式错误
phone/^1[3-9]\d{9}$/手机号格式错误
email/^[^\s@]+@[^\s@]+\.[^\s@]+$/邮箱格式错误
number/^-?\d+(\.\d+)?$/请输入有效数字
url/^https?:\/\/.+/网址格式错误
required/.+/此项不能为空
custom由 data-validate-pattern 指定输入格式不正确

验证行为规则

  • 触发时机：输入框失焦（blur）时执行验证
  • 错误清除：用户继续输入（input 事件）时自动清除错误状态
  • 空值策略：
    • required 规则：空值 触发错误
    • 其他规则（idcard/phone/email/number/url/custom）：空值 不触发错误（仅在填写时校验格式）
  • 无 data-validate 属性：不进行验证，任何值均视为有效

CUI-input 验证示例

普通输入框验证时添加 CUI-input--error 类（红色边框），错误信息通过 title 属性显示（鼠标悬停可看）：

<input type="text" name="idcard" class="CUI-input" data-validate="idcard" data-error="自定义错误信息">
<input type="text" name="phone" class="CUI-input" data-validate="phone">
<input type="text" name="email" class="CUI-input" data-validate="email">

CUI-input-box 验证示例

组合框验证时添加 CUI-is-error 类，同时显示 .CUI-input__error > .CUI-message-error 中的错误信息文字：

<input type="text" name="phone" class="CUI-input-box" data-label="手机号" data-validate="phone" data-hint="提示文字" data-error="错误信息">
<input type="text" name="requiredField" class="CUI-input-box" data-label="必填" data-validate="required" data-hint="必填项" data-error="请填写">

自定义正则验证

使用 data-validate="custom" 搭配 data-validate-pattern 指定正则：

<input type="text" name="code" class="CUI-input" data-validate="custom" data-validate-pattern="^\d{6}$" placeholder="6位数字">

错误状态 CSS 类

类名适用容器效果
CUI-input--error.CUI-input红色边框 + 红色阴影 + 图标变红
CUI-is-error.CUI-input-box显示内部的 .CUI-input__error 错误信息块

注意：错误状态完全由 CSS 类驱动，不修改 data 属性。验证引擎通过 classList.add/remove 控制显示，与用户自定义样式无冲突。

表单验证

表单内的输入框自动按 form.id.name 路径注册。表单级的批量验证 API：

API说明返回值
CUI.input.isValid(path) 检查单个输入框是否有效 boolean — true 表示值有效
CUI.input.isFormValid(formId) 检查整个表单是否全部有效 boolean — true 表示所有字段均通过验证 
// 表单内所有字段正确时才通过
if (CUI.input.isFormValid('myForm')) {
    // 提交表单...
} else {
    // 有错误字段，框架已自动高亮并显示错误信息
}

属性保留（Property Preservation）

CUI-input-box 渲染时会创建一个新的 <input> 元素替代原始元素。以下属性会被自动拷贝到新 input 上：

属性说明
id元素 ID，保留以便 document.getElementById 访问
readonly只读状态
disabled禁用状态
requiredHTML5 原生必填标记
autocomplete自动完成行为
inputmode移动端键盘模式
maxlength最大字符数
min最小值
max最大值
step步进值
patternHTML5 原生正则模式
data-validate框架验证规则
data-validate-pattern框架自定义验证正则

注意：data-error、data-label、data-hint、data-info、data-icon、data-isError 等已被框架消费的属性不会被拷贝到新 input 上。

手动初始化

在某些场景下（如 AJAX 加载内容后），可手动调用初始化：

// 重新扫描 DOM 并注册新元素
CUI.inputInit();

注意：已处理的元素不会重复处理，processedNames 和 beautifiedElements 提供了双重防重复保护。

事件系统

框架使用事件委托模式处理所有输入框交互，而非逐个绑定：

事件阶段行为
blur 捕获阶段（capture） 触发验证：读取 data-validate 规则，校验值，更新 node.valid 和 CSS 类
input 冒泡阶段 清除错误状态（移除 CSS 类，重置 node.valid = true），同步值到注册表

采用事件委托的优势：无论动态添加多少输入框，只需两个全局监听器，性能恒定 O(1)。

自动注册与生命周期

DOM 扫描

  • 初次加载：通过 scanInputs() 全量扫描 input/textarea/select 元素
  • 动态添加：通过 CastingDOMObserver 自动检测新增元素并处理
  • 移除清理：元素移除时，自动从注册表注销

防重复机制

  • processedNames（Set）：按 name 标记已注册的输入框，防止重复注册
  • beautifiedElements（WeakSet）：按 DOM 元素引用标记已美化的输入框，防止重复渲染

CUI-input-box 的 DOM 重建

CUI-input-box 渲染时会将原始 <input> 替换为新的包装结构。为了确保开发者能继续正常访问元素：

  • id 被拷贝到新 input 上
  • 所有原生 HTML 属性（readonly、disabled、maxlength 等）被拷贝
  • 框架验证属性（data-validate、data-validate-pattern）被拷贝
  • 已消费的 data 属性（data-label、data-hint、data-error、data-icon、data-info、data-isError）不会被拷贝
  • 注册表的 element 指向新创建的 input

模块加载说明

input 模块支持两种加载方式：

方式适用场景示例
通过框架入口（推荐） 使用完整 Casting UI 框架 <script type="module" src="/modules/js/index.js">
独立加载 仅需 Input 功能 <script src="/modules/js/input.js">

独立加载时，模块会自动检测调度器是否可用，不可用时直接初始化。
