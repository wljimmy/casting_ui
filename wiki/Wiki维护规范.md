## Markdown 格式规范

### 基本规范
1. **文件编码**: UTF-8
2. **换行**: LF
3. **缩进**: 2 空格
4. **标题层级**: 从 H2 (`##`) 开始，H1 仅用于文档标题

### 标题格式
```markdown
# 文档标题（一级标题，仅一个）

## 章节标题（二级标题）

### 小节标题（三级标题）

#### 子小节标题（四级标题）
```

### 列表格式

#### 无序列表
```markdown
- 项目一
- 项目二
  - 子项目
  - 子项目
```

#### 有序列表
```markdown
1. 第一步
2. 第二步
3. 第三步
```

#### 检查列表
```markdown
- [x] 已完成项
- [ ] 待完成项
```

### 表格格式

#### 标准表格
```markdown
| 列标题一 | 列标题二 | 列标题三 |
|---------|---------|---------|
| 内容一   | 内容二   | 内容三   |
| 内容四   | 内容五   | 内容六   |
```

#### 带对齐的表格
```markdown
| 左对齐 | 居中 | 右对齐 |
| :----- | :--: | -----: |
| 文本一 | 文本二 | 文本三 |
```

### 链接格式

#### 内部链接
```markdown
[链接文本](./相对路径/文件.md)
[首页](./Home.md)
[组件文档](./Components/index.md)
```

#### 外部链接
```markdown
[链接文字](https://example.com)
```

#### 锚点链接
```markdown
[跳转到章节](#章节标题)
```

### 代码格式

#### 行内代码
```markdown
使用 `代码` 标签
```

#### 代码块
```markdown
```语言
代码内容
```
```

常用语言标识：`html`、`css`、`javascript`、`bash`、`markdown`

### 强调格式
```markdown
**粗体文本**
*斜体文本*
~~删除线文本~~
```

### 引用格式
```markdown
&gt; 引用文本
&gt; 多行引用
```

## 新增文档流程

### Step 1: 确定文档位置
根据内容类型确定存放目录：
- 组件文档 → `Components/`
- 核心模块 → `Core/`
- API 文档 → `API/`
- 通用指南 → 根目录

### Step 2: 创建文档文件
使用对应模板创建新文档，确保：
- [ ] 文件命名符合规范
- [ ] 包含完整文档结构
- [ ] 使用正确的 Markdown 格式

### Step 3: 编写内容
1. 参考现有手册页面 (`public/manual/`) 的内容
2. 查阅相关源代码 (`src/modules/`) 确保准确性
3. 添加实用的代码示例
4. 确保与现有文档风格一致

### Step 4: 更新索引
在相关目录的 `index.md` 中添加新文档的链接。

### Step 5: 更新首页导航
如果是重要文档，更新 `Home.md` 中的导航。

### Step 6: 验证检查
- [ ] 所有内部链接正确
- [ ] 代码示例正确
- [ ] Markdown 格式无误
- [ ] 与现有文档风格一致

## 修改文档流程

### Step 1: 理解修改需求
1. 明确修改范围：是修正错误、更新内容、还是新增章节？
2. 确认修改依据：参考手册页面、源代码或用户反馈

### Step 2: 备份（可选但推荐）
检查 Git 状态，确保可以回滚。

### Step 3: 执行修改
1. 仅修改必要的内容
2. 保持文档整体风格一致
3. 更新相关的代码示例
4. 如有链接变更，同步更新所有引用

### Step 4: 验证修改
- [ ] 修改内容准确无误
- [ ] 链接仍然有效
- [ ] 格式保持一致
- [ ] 没有引入新的错误

## 代码示例规范

### HTML 示例

#### 基础示例
```html
&lt;!-- 正确：完整可运行 --&gt;
&lt;button class="btn btn-primary"&gt;按钮&lt;/button&gt;
```

#### 完整页面示例
```html
&lt;!DOCTYPE html&gt;
&lt;html&gt;
&lt;head&gt;
  &lt;title&gt;示例&lt;/title&gt;
  &lt;link rel="stylesheet" href="src/modules/css/core.css"&gt;
&lt;/head&gt;
&lt;body&gt;
  &lt;div class="CUI-section"&gt;
    &lt;!-- 内容 --&gt;
  &lt;/div&gt;
  &lt;script type="module" src="src/modules/js/index.js"&gt;&lt;/script&gt;
&lt;/body&gt;
&lt;/html&gt;
```

### JavaScript 示例

#### API 调用示例
```javascript
// 正确：完整示例
window.CUI.input.getValue('form-name');

// Promise 示例
window.openColorPicker().then(color =&gt; {
  console.log(color);
}).catch(() =&gt; {
  console.log('取消');
});
```

### 注释规范
- 代码中添加简洁注释说明关键部分
- 示例代码保持简洁，避免冗余
- 复杂示例提供多行注释

### 示例原则
- ✅ **可运行**: 示例代码应该是完整的、可以直接运行的
- ✅ **清晰**: 示例应该清晰展示功能
- ✅ **适度**: 不要提供过多示例，重点突出最常用的用法

## 链接和导航维护

### 内部链接规范
- 使用相对路径，避免绝对路径
- 指向文件时包含扩展名 `.md`
- 链接文本应该有描述性

```markdown
&lt;!-- 正确 --&gt;
[首页](./Home.md)
[组件文档](./Components/index.md)

&lt;!-- 错误 --&gt;
[首页](Home.md)          // 缺少 ./
[首页](./Home)            // 缺少扩展名
[点这里](./Home.md)       // 描述不清晰
```

### 导航更新场景
需要更新导航的情况：
1. 新增重要文档时
2. 删除或移动文档时
3. 文档重命名时

### 索引维护
- 每次新增/删除文档时，更新对应目录的 `index.md`
- 保持索引列表有序、分类清晰

## 风格参考

### 语气风格
- 清晰、简洁、准确
- 避免使用过于口语化的表达
- 使用用户友好的语言
- 提供足够的上下文说明

### 文档语气示例

**好**:
```markdown
使用 `data-type` 属性配置组件类型。
```

**不好**:
```markdown
你可以用这个属性来改，就像这样...
```

### 中文规范
- 使用标准中文标点符号
- 代码和中文之间加空格
- 专有名词使用标准翻译或保留英文

```markdown
&lt;!-- 正确 --&gt;
使用 `btn-primary` 类创建主要按钮

&lt;!-- 错误 --&gt;
使用`btn-primary`类创建主要按钮
```
