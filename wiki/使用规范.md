## 文件引入规范

### 方式一：使用合并文件（生产环境推荐）
```html
<!-- 引入 CSS -->
<link rel="stylesheet" href="dist/v{版本号}/Casting.css">

<!-- 引入 JavaScript -->
<script src="dist/v{版本号}/Casting.js"></script>
```

### 方式二：使用模块化文件（开发环境）
```html
<!-- CSS 模块 -->
<link rel="stylesheet" href="src/modules/css/core.css">
<link rel="stylesheet" href="src/modules/css/container.css">
<link rel="stylesheet" href="src/modules/css/layout.css">
<link rel="stylesheet" href="src/modules/css/components.css">
<!-- 按需引入其他 CSS -->

<!-- JavaScript 模块 -->
<script type="module" src="src/modules/js/index.js"></script>
```

### 注意事项
- ✅ **推荐**: 生产环境使用合并文件以减少请求
- ✅ **允许**: 开发环境使用模块化文件便于调试
- ❌ **禁止**: 混合使用合并文件和模块文件
- ❌ **禁止**: 修改 dist/ 目录下的文件，修改应在 src/ 中进行

## HTML 编写规范

### 基础原则
1. **使用原生 HTML 标签** - 不使用自定义标签
2. **使用语义化标签** - `<header>`, `<nav>`, `<main>`, `<footer>` 等
3. **通过 data-* 属性配置** - 不使用框架专属属性

### 正确示例
```html
<!-- 容器 -->
<div class="CUI-section">
  <div class="CUI-box">内容</div>
</div>

<!-- 按钮 -->
<button class="btn btn-primary">按钮</button>

<!-- 菜单 -->
<menu>
  <ul>
    <li data-icon="home">首页</li>
  </ul>
</menu>

<!-- 输入框 -->
<input type="text" name="username" class="CUI-input" data-label="用户名">
```

### 错误示例
```html
<!-- ❌ 不要这样 -->
<CastingButton></CastingButton>
<div cui-option="xxx"></div>
<input custom-prop="value">
```

## CSS 使用规范

### CSS 变量使用
```css
/* ✅ 正确 - 使用 CSS 变量自定义主题 */
:root {
  --primary-color: #165DFF;
  --bg-color: #ffffff;
  --text-primary: #333333;
}

/* ❌ 错误 - 不要直接覆盖框架类样式 */
.btn-primary {
  background-color: red !important; /* 避免 */
}
```

### 自定义样式
```css
/* ✅ 正确 - 使用自定义类名 */
.my-custom-btn {
  padding: 10px 20px;
}

/* ✅ 正确 - 使用主题管理器 */
/* 使用 openThemeSelector() 来管理主题 */
```

### 样式优先级
1. **CSS 变量** - 最高优先级，推荐
2. **自定义类** - 其次推荐
3. **内联样式** - 仅在特殊情况下使用
4. **!important** - 尽量避免使用

## JavaScript 调用规范

### 全局对象使用
```javascript
// ✅ 正确 - 使用框架提供的全局对象
window.CUI.input.getValue('form.field');
window.openColorPicker();
window.openThemeSelector();
window.showToast('success', '消息');

// ✅ 正确 - ES6 模块导入（开发环境）
import { openColorPicker } from './color-picker.js';
```

### API 调用约定
```javascript
// ✅ 正确 - Promise 处理
openColorPicker().then(color => {
  console.log('选择的颜色:', color);
}).catch(() => {
  console.log('用户取消');
});

// ✅ 正确 - async/await
async function pickColor() {
  try {
    const color = await openColorPicker();
    console.log(color);
  } catch (error) {
    console.log('取消选择');
  }
}
```

### DOM 事件监听
```javascript
// ✅ 正确 - 使用框架的 DOMObserver
window.CastingDOMObserver.onAdd('my-component', '.my-component', (el) => {
  // 初始化逻辑
});

// ✅ 正确 - 内联事件（用于用户自定义）
<button onclick="handleClick()">按钮</button>

// ⚠️ 注意 - 避免在手册页和示例页中使用 addEventListener
// 如需使用，确保在页面卸载时清理
```

## 组件使用规范

### 组件初始化
- ✅ **自动初始化**: 框架会自动检测并初始化组件
- ✅ **data-* 配置**: 使用 data 属性配置组件
- ❌ **手动调用**: 除非特殊情况，不需要手动调用初始化函数

### 组件命名约定
- **容器**: `CUI-*` 前缀 (CUI-box, CUI-card, CUI-grid)
- **按钮**: `btn-*` 前缀 (btn-primary, btn-secondary)
- **状态**: `*-success`, `*-warning`, `*-error`, `*-info`

### 组件数据获取
```javascript
// ✅ 正确 - 使用注册表
const data = window.CUI.input.getFormData('my-form');

// ✅ 正确 - 使用路径访问
const value = window.CUI.input.getValue('form.username');
```

## 目录组织规范

### 项目目录结构
```
Casting_UI/
├── src/                    # 源代码（开发用）
│   ├── modules/           # 框架模块
│   │   ├── js/           # JavaScript 模块
│   │   └── css/          # CSS 模块
│   ├── examples/         # 示例页面
│   └── index.html        # 首页
├── public/               # 静态资源
│   ├── manual/          # 手册页面
│   └── icons/           # 图标资源
├── dist/                # 发布文件（不要直接修改）
│   └── v{版本号}/       # 版本化发布
├── docs/                # 文档
├── test/                # 测试
└── wiki/                # Wiki 文档（本目录）
```

### 文件放置原则
| 内容 | 位置 |
|------|------|
| 框架源代码 | `src/modules/` |
| 组件示例 | `src/examples/` |
| 手册页面 | `public/manual/` |
| 发布文件 | `dist/v{版本号}/` |
| Wiki 文档 | `wiki/` |
| 测试页面 | `test/` |

## 禁止事项

### ❌ 禁止修改原始文件
- 不要直接修改 `dist/` 下的发布文件
- 不要修改 `public/manual/` 中的手册页框架部分
- 修改应在 `src/modules/` 中进行

### ❌ 禁止破坏框架机制
- 不要覆盖或修改 `window.CUI` 命名空间
- 不要删除或修改 DOMObserver 相关代码
- 不要修改核心模块的导出

### ❌ 禁止混用开发/生产模式
- 不要同时引入合并文件和模块文件
- 不要在生产环境使用模块化文件
- 不要在开发环境直接使用 dist 文件

## 最佳实践

1. **版本锁定** - 在项目中锁定使用的框架版本
2. **测试先行** - 升级前在测试环境验证
3. **文档同步** - 使用新功能后及时更新项目文档
4. **渐进式采用** - 可以部分使用框架，逐步迁移
5. **反馈贡献** - 发现问题或建议及时反馈
